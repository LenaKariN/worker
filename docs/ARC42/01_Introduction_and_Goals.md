## 1. Введение и Цели

### 1.1. Концепция системы
Приложение, позволяющее создавать, изменять и просматривать организационную структуру некоторой компании.
В оргструктуру входят **Сотрудники**, **Организационные Единицы** (департаменты, отделы, группы) 
произвольной структуры (древовидной, матричной), а так же типы отношений между **Сотрудниками**, 
**Организационными Единицами** и друг с другом.
Типы отношений, типы Позиций (должностей) и типы Организационных Единиц можно так же создавать и изменять.
Приложение учитывает историчность изменений оргструктуры, а не только ее последнюю версию.
Физическое удаление экземпляров сущностей не предусмотрено, предусмотрено "мягкое" удаление.

- Информация о Сотрудниках должна содержать ФИО, контактные данные, зарплату.  
- Сотрудник может работать в Организации, которая состоит из департаментов, департаменты из отделов, отделы из команд и т.п.
- Сотрудник может иметь подчиненных и сам быть подчиненным у других.  
- Сотрудник может менять место в организационной структуре, причем в ограниченном интервале даты/времени.  
- Организационная структура не может быть циклической (например, нельзя быть начальником собственного начальника).

### 1.2. Обзор требований

| №      | Требование                                                                                                             | Мотивация и комментарии              |
|--------|------------------------------------------------------------------------------------------------------------------------|--------------------------------------|
| `FR01` | Создать/изменить/удалить/просмотреть `Сотрудника`                                                                      | Требование "Индивидуального задания" |    
| `FR02` | Просмотреть список `Сотрудников`                                                                                       | Требование "Индивидуального задания" |    
| `FR03` | Просмотреть список `Сотрудников` со стажем > N лет                                                                     | Требование "Индивидуального задания" |    
| `FR04` | Создать/изменить/удалить/просмотреть элементы справочника `Позиций Сотрудника`                                         | Дополнительные обязательства         |    
| `FR05` | Создать/изменить/удалить/просмотреть `Организационную Единицу`                                                         | Дополнительные обязательства         |    
| `FR06` | Создать/изменить организационную структуру (связи `Организационных Единиц` и `Сотрудников`)                            | Дополнительные обязательства         |    
| `FR07` | Просмотреть организационную структуру выбранной `Организационной Единицы` или `Сотрудника` на указанный период времени | Дополнительные обязательства         |

| №       | Цель                                         | Мотивация и комментарии                                                 |
|---------|----------------------------------------------|-------------------------------------------------------------------------|

| `NFR02` | Удобство использования и эстетика            | Демонстрация проекта должна вызывать приятные эмоции                    |

| `NFR04` | Легкость расширения функциональности системы | При доработках хочется менять только небольшую изолированную часть кода |
| `NFR05` | Строгое поддержание целостности данных       | Должны выполняться бизнес-правила для данных Сотрудника и т.п.          |
