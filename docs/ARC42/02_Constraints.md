## 2. Ограничения

### 2.1. Технические ограничения

| №      | Ограничения                              | Пояснения и предыстория                                           |
|--------|------------------------------------------|-------------------------------------------------------------------|
| `TC01` | Тип приложения - `CLI`                   | Основываясь на кейс-задаче №3 "Индивидуального задания"           |
| `TC02` | Хранение данных в оперативной памяти     | Основываясь на кейс-задаче №3 "Индивидуального задания"           |
| `TC03` | Язык программирования - `Python`         | Наиболее удобный и простой для CLI                                |
| `TC04` | Язык описания требований - `Gherkin`     | Удобный способ описать приемочные критерии на руссом языке        |

### 2.2. Организационные ограничения

| №      | Ограничения                 | Пояснения и предыстория                                                                                                                                                                       |
|--------|-----------------------------|-----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------|
| `OC01` | Ограниченный учебный проект | Минимальные исходные требования для реализации проекта получены из Учебного Задания                                                                                                           |
| `OC02` | Расширенный учебный проект  | Полный перечень требований включает минимальные (`OC01`) и дополнительные, принятые с целью изучить на практике аналитический паттерн "Accountability" из книги "Analysis Patterns" M.Fowler. |

### 2.3. Соглашения

| №     | Соглашение                                   | Пояснения и предыстория                                                        |
|-------|----------------------------------------------|--------------------------------------------------------------------------------|
| `C01` | Инкрементальная разработка                   | Из-за недостатка времени необходимо вести разработку инкрементально            |
| `C02` | Комментарии по-русски                        | Исходный код будут читать только русскоговорящие                               |
| `C03` | Соглашение о коммитах (Conventional Commits) | Основано на [данном документе](https://www.conventionalcommits.org/ru/v1.0.0)  |
| `C04` | Семантическое версионирование (SemVer)       | Важно для инкрементальной разработки и автоматически управляется `C03`         |
